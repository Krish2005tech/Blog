How I Set Up This Blog (My Journey from Idea ‚Üí Deployment)

A detailed walkthrough of how I built this site using Hugo + Cloudflare Pages, the decisions I made, the struggles I faced, and why I chose this stack.
December 11, 2025

For a long time, I wanted a simple place on the internet where I could write about tech, document small wins, share problems I solve, or even drop personal thoughts. Not a fancy platform, not something overloaded with features ‚Äî just a clean space where I could write my way, at my pace.

üß† My Mindset: Keep It Simple, But Developer-Friendly#

I knew I wanted a few things right from the start:

  • Minimal setup ‚Äî because I don’t want to spend weeks installing plugins.

Before jumping in, I looked at several options:

üß± The Actual Steps I Took (So You Can Do It Too)#

Here’s exactly how I did it:

1. Install Hugo#

On Windows:

choco install hugo

Check:

hugo version

2. Create a new Hugo site#

hugo new site Blog
cd Blog

3. Add PaperMod theme#

git init
git submodule add https://github.com/adityatelange/hugo-PaperMod.git themes/PaperMod

Edit hugo.toml to include:

theme = "PaperMod"

4. Create first pages#

Homepage content, About page, and first post.

Example:

hugo new posts/hello-world.md

I edited them later to add proper text.

5. Preview locally#

hugo server -D

This was the moment where I finally saw my site.

6. Push to GitHub#

git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin <repo-url>
git push -u origin main

7. Deploy to Cloudflare Pages#

This part was trickier.

At first, I accidentally created a Worker instead of a Pages project#

This resulted in Cloudflare ignoring Hugo entirely and showing only:

Hello world

And later a 404 page.

I spent time debugging it until I realized:

üëâ I must use Pages, not Workers.

So I deleted the Worker and created a Pages ‚Üí Connect to Git project.

Inside Cloudflare Pages, I set:

  • Build command: 
    hugo
  • Output directory: 
    public
  • Env variable: 
    HUGO_VERSION = 0.152.2

Then Cloudflare finally ran a real build.

8. Connecting my .tech domain#

When my domain became available, I added:

blog.krish2005.tech

Cloudflare automatically configured DNS and HTTPS.

Seeing my site live on a real domain was so satisfying.

üß© Problems I Faced & How I Solved Them#

‚ùå Problem 1 ‚Äî Cloudflare kept showing “Hello world”#

This was because it was deploying as a Worker, not a Page.

Fix: Delete Worker ‚Üí Create Pages project.

‚ùå Problem 2 ‚Äî 404 after deployment#

This happened because Cloudflare deployed 0 files.

Fix: Ensure output directory = public And Hugo actually builds properly.

‚ùå Problem 3 ‚Äî Couldn’t change .pages.dev URL#

Turns out: Page subdomains are fixed. Solution: use my custom .tech domain instead.

üéâ What I Learned#

  • Cloudflare Pages is one of the best free dev tools available today.
  • A blog doesn’t have to be perfect ‚Äî it just needs to exist.
  • Writing is easier when you own your space.
